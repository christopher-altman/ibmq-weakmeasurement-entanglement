# IBM Weak Measurement Entanglement Active Design

**Thesis:** Adaptive weak-measurement experiment design with conformal-calibrated uncertainty for concurrence (and negativity), with IBM Quantum hardware validation via Qiskit Runtime against simulation and tomography baselines.

## Overview
This repository implements a qubit-native extension of weak-value entanglement inference: a 3-qubit experiment (`A,B` system + pointer ancilla `P`) with tunable weak coupling, active/adaptive setting selection, and finite-sample uncertainty calibration. The benchmark family is
\[
\rho_{AB}(p,\theta)=p|\psi_\theta\rangle\langle\psi_\theta|+(1-p)\frac{I_A}{2}\otimes\rho_B^\theta,
\quad |\psi_\theta\rangle=\cos\theta|00\rangle+\sin\theta|11\rangle,
\]
with primary target concurrence and secondary target negativity.

## Weak Measurement on IBMQ
### Circuit sketch

```text
q0: A  --[state prep]------H------M_A---------------------
q1: B  --[state prep]--o----(optional R_y(phi_B))--M_B----
                     | 
q2: P  --|0>---------[U(g)=exp(-i g X_B⊗Y_P /2)]--R_basis--M_P

U(g) decomposition (transpilation-friendly):
  basis change on B,P -> CX(B,P) -> RZ(g) on P -> CX(B,P) -> undo basis change

Pointer basis readout:
  Z: I
  X: H
  Y: Sdg then H
```

Postselection is applied in analysis: keep `A=0` (equivalent to `|+>` before `H`), then condition on `B=k`.

## Quickstart

### Install
```bash
pip install -e .
```

### Reproduce simulation artifacts
```bash
python -m src.main \
  --mode sweep \
  --backend sim \
  --noise ideal \
  --policy adaptive \
  --g_grid 0.00,0.10,0.20,0.35,0.50,0.70,0.90 \
  --shots 2000 \
  --seed 7 \
  --export_dir artifacts
```

### Run on IBM Quantum hardware (Runtime SamplerV2)
```bash
export QISKIT_IBM_TOKEN="YOUR_IBM_QUANTUM_TOKEN"
python -m src.main \
  --mode hardware_run \
  --backend ibm \
  --ibm_backend_name ibm_kyoto \
  --policy adaptive \
  --g_grid 0.10,0.20,0.35,0.50 \
  --shots 4000 \
  --seed 7 \
  --export_dir artifacts
```

### Hardware dry run (no submission)
```bash
python -m src.main \
  --mode hardware_run \
  --backend ibm \
  --ibm_backend_name ibm_kyoto \
  --policy adaptive \
  --g_grid 0.10,0.20,0.35,0.50 \
  --shots 4000 \
  --seed 7 \
  --dry_run \
  --export_dir artifacts
```

## Interpretation
The adaptive policy should reduce median shots needed to reach target concurrence error and should remain useful under noise/hardware shift. Conformal intervals provide an explicit reliability layer: when shift/uncertainty grows, intervals widen and abstention can trigger.

## Reproducibility Checklist
- Fixed seeds: Python, NumPy, Torch.
- Deterministic simulator sampling and deterministic adaptive tie-breaks.
- Environment dump written into `artifacts/summary.md`.
- IBM jobs and counts cached in `artifacts/ibm_cache/`.
- Tests:
```bash
python -m pytest
```

## Why this matters
Weak-measurement entanglement inference is attractive because it compresses global information into local postselected statistics. In practice, however, deployment requires decisions under finite shots and nonstationary hardware. This repository adds active information-seeking and finite-sample calibration so the method can be evaluated as an engineering-grade estimator, not only as a point predictor.
